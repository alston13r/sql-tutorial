Module 8

GROUP BY & HAVING

All exercises in this lesson have been completed.

Module 5 introduced aggregate functions that summarize an entire table. This module shows how to run those same calculations per group — for example, average salary in every department with a single query.

Recap: one department at a time

In Module 5, you found the average salary in Finance like this: 

SELECT AVG(salary) FROM employees WHERE department = 'Finance';

That works for one department, but repeating it for every department means writing many queries. GROUP BY collapses rows that share the same value and lets you aggregate each group separately.

The practice database has 14 employees across 8 departments. Frank Lee in HR has a NULL salary, which still affects some grouped averages.

GROUP BY

GROUP BY splits rows into groups — usually by a column such as department — then runs an aggregate function on each group. Every non-aggregated column in the SELECT list must appear in the GROUP BY clause.

To find the average salary in each department: 

SELECT department, AVG(salary) AS avg_salary
FROM employees
GROUP BY department;

Result: 8 rows — one per department. HR's avg_salary is NULL because Frank Lee's salary is unknown.

HAVING

WHERE filters individual rows before grouping. HAVING filters groups after aggregation. You cannot put aggregate functions like COUNT(*) in a WHERE clause — use HAVING instead.

To list only departments with at least two employees: 

SELECT department, COUNT(*) AS employee_count
FROM employees
GROUP BY department
HAVING COUNT(*) >= 2;

Result: 5 rows — Engineering, Marketing, Sales, Finance, and Support.

Summary

Grouped queries follow a clear order of operations:

  • WHERE — filter individual rows before they are grouped
  • GROUP BY — collapse rows into groups; one result row per distinct group value
  • Aggregates — COUNT, AVG, SUM, etc. run on each group
  • HAVING — filter groups after aggregation (e.g. only groups where COUNT(*) >= 2)

You can combine them in one query — for example, filter rows with WHERE, group with GROUP BY, then keep only the groups you care about with HAVING.
