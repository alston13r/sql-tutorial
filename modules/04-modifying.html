Module 4

Modifying Data

All exercises in this lesson have been completed.

So far, your queries have only read data. This module introduces commands that change what is stored in a table: adding rows, updating values, and deleting rows.

Recap: reading vs changing data

SELECT displays data without changing the table. The commands below modify the database. Use Run to preview table contents after a change, and Reset to reload the original seed data.

INSERT INTO

INSERT INTO adds a new row. List the columns you are filling in, then provide matching values in a VALUES clause.

For this first exercise, the sandbox uses the simplified employees table from Module 1 — just id, name, department, salary, and hired_date. Later exercises use a larger table with extra columns like job_title.

Example: add a new HR employee with id 7: 

INSERT INTO employees (id, name, department, salary, hired_date)
VALUES (7, 'Jordan Ellis', 'HR', 63000, '2024-02-01');

Exercise

On the simplified employees table, insert a new row with id 7: Riley Shaw, department Operations, salary 67000, hired 2024-01-08. Use Check when done; Reset restores the seed data.

UPDATE

UPDATE changes existing rows. Use SET to name the columns to change and always include WHERE so you only affect the rows you intend.

The sandboxes below use the extended employees table (with job_title). Example: give Bob Martinez a raise to 90,000: 

UPDATE employees SET salary = 90000 WHERE name = 'Bob Martinez';

DELETE

DELETE removes rows from a table. Without a WHERE clause, every row is deleted — so be careful.

Example: remove the employee with id 14: 

DELETE FROM employees WHERE id = 14;

NULL Values

NULL means missing or unknown — it is not the same as 0 or an empty string. Use IS NULL and IS NOT NULL to test for it (not = NULL).

In the extended practice database, Frank Lee already has an unknown salary: 

SELECT name, salary FROM employees WHERE salary IS NULL;

UPDATE employees SET salary = NULL WHERE name = 'Frank Lee';

Exercise

Set Noah Taylor's salary to NULL (unknown).
