Module 10

Null Handling

All exercises in this lesson have been completed.

Earlier modules showed what NULL means and how it affects filters, joins, and aggregates. This module focuses on SQLite functions that replace NULL values with something more useful in your output.

Recap: testing vs replacing

Use IS NULL when you want to test whether a value is missing. Use IFNULL or COALESCE when you want to replace that missing value in the query result.

The practice database still uses Frank Lee's unknown salary as the main example.

IFNULL()

IFNULL(value, fallback) returns the original value when it is not NULL. Otherwise, it returns the fallback value. It is the simplest SQLite function for replacing missing data.

To show employee salaries and replace unknown salaries with 0: 

SELECT name, IFNULL(salary, 0) AS salary_display
FROM employees;

Frank Lee's salary appears as 0 in the result. Everyone else keeps their real salary value.

COALESCE()

COALESCE(a, b, c, ...) returns the first value in the list that is not NULL. With two arguments it behaves like IFNULL, but it becomes more flexible when you want to try multiple fallback values from left to right.

To show a fallback label for missing salaries: 

SELECT name, COALESCE(salary, 'Unknown') AS salary_label
FROM employees;

With two arguments, COALESCE behaves like IFNULL. Frank Lee's salary becomes Unknown because his stored salary value is NULL.

Summary

SQLite gives you a couple of simple ways to replace missing values:

  • IFNULL(x, y) is best when you only need one fallback value.
  • COALESCE(a, b, c, ...) is best when you want SQL to try several values and return the first one that is not NULL.
  • Use IS NULL to test for missing data, and use IFNULL or COALESCE to replace it in the result.
