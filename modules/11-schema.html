Database & Tables

All exercises in this lesson have been completed.

This module focuses on schema definition: creating tables, choosing column types, changing table structure, and dropping tables you no longer need.

Recap: schema vs data

Most earlier modules worked with rows already stored in a table. Here, you are changing the structure of the database itself. That means using DDL commands such as CREATE TABLE, ALTER TABLE, and DROP TABLE.

In SQLite, the database is just a file, so this lesson focuses on creating and changing tables. The next module adds rules about what data those tables should allow.

Data Types

Before creating a table, you need to decide what kind of data each column will store. In this module, the most common SQLite types are INTEGER for whole numbers, TEXT for strings, and REAL for decimal values.

You will also often see dates stored as TEXT in SQLite, especially when using a format like YYYY-MM-DD.

Choosing a sensible type helps make the table structure clear before you start inserting data or adding constraints in the next module.

CREATE TABLE and Data Types

CREATE TABLE defines a new table and its columns. Once you know the column names and types you need, you can describe the structure directly in the table definition.

For example, here is a table shaped like the expanded employees table used in other modules: 

CREATE TABLE employees (
  id INTEGER,
  name TEXT,
  department TEXT,
  job_title TEXT,
  salary REAL,
  hired_date TEXT
);

This defines the table structure only. It does not insert any rows yet.

ALTER TABLE

ALTER TABLE changes an existing table. In SQLite, common operations include renaming a table, renaming a column, and adding a new column.

To add a new column to the projects table: 

ALTER TABLE projects ADD COLUMN budget REAL;

This keeps the existing rows and adds the new column to the table definition.

DROP TABLE

DROP TABLE removes a table completely, including its schema and data. It is destructive, so you should be sure you no longer need the table before dropping it.

Example: 

DROP TABLE old_logs;

After the command runs, the table is gone from the database.

Summary

Schema commands change the structure of a database:

  • CREATE TABLE defines a new table and its columns.
  • ALTER TABLE changes an existing table definition.
  • DROP TABLE removes a table completely.

In this module, you defined and changed table structure. The next module builds on that by adding rules about what data the schema should allow.
