Module 3

Filtering & Sorting

All exercises in this lesson have been completed.

So far, your queries return every row in a table. This module shows how to filter rows with WHERE, combine conditions, and sort results with ORDER BY.

Recap: reading data

A basic SELECT returns all rows from a table. To narrow the results, add a WHERE clause with a condition: 

SELECT name, department FROM employees;

This shows every employee. The sections below explain how to keep only the rows you care about.

WHERE

The WHERE clause filters rows. Only rows that meet the condition are included in the result. You can compare columns to values using operators like =, >, and <.

To find employees in the Engineering department: 

SELECT name, department FROM employees WHERE department = 'Engineering';

Text values like department names must be wrapped in single quotes.

AND, OR

You can combine conditions with AND and OR. AND requires every condition to be true; OR requires at least one.

To find Engineering employees who earn more than 80,000: 

SELECT name, salary FROM employees
WHERE department = 'Engineering' AND salary > 80000;

When you mix AND and OR in one query, use parentheses to make your intent clear.

NOT

The NOT keyword reverses a condition, excluding rows that would otherwise match.

To list employees who are not in HR: 

SELECT name, department FROM employees WHERE NOT department = 'HR';

This is equivalent to department != 'HR'. Either form is fine as long as the result is correct.

ORDER BY

By default, query results are not guaranteed to be in any particular order. The ORDER BY clause sorts rows by one or more columns.

To list all employees sorted by salary, highest first: 

SELECT name, salary FROM employees ORDER BY salary DESC;

DESC sorts from highest to lowest. Use ASC (the default) for lowest to highest. This is when row order finally matters — unlike earlier modules where only the set of rows counted.

Sorting by multiple columns

When two rows have the same value in the first sort column, SQLite does not guarantee which comes first. Add another column to ORDER BY to break ties — and you can list as many columns as you need, not just two.

List employees by department, then alphabetically by name within each department: 

SELECT name, department FROM employees
ORDER BY department ASC, name ASC;

Columns are evaluated left to right: sort by department first, then use name only when the department matches. If those still tie, you could add a third column (such as salary) to sort further.
