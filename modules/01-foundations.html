Introduction to SQL & SQLite

All exercises in this lesson have been completed.

SQL (Structured Query Language) is the standard language for working with relational databases. In this tutorial, you'll use SQLite to explore data, answer questions, and gradually build confidence with queries.

What you'll learn

  • What a relational database and table are
  • How rows and columns organize information
  • Basic SQL syntax: statements end with ;
  • How to write a SELECT query to read data

What is a table?

A database stores information in tables. You can think of a table like a spreadsheet: it has named columns across the top and rows of data underneath.

In this lesson, you'll work with an employees table. Each row represents one employee, and each column tells you something about that employee, such as their name, department, salary, or hire date.

Rows and columns

A column describes one type of information. In the employees table, the columns are id, name, department, salary, and hired_date.

A row is one complete record. For example, Alice Chen is one row in the table because her ID, name, department, salary, and hire date are all stored together.

Example: the employees table

Here is a sample of the data loaded for this lesson:

id name department salary hired_date
1 Alice Chen Engineering 95000 2021-03-15
2 Bob Martinez Engineering 88000 2020-07-01
3 Carol Williams Marketing 72000 2022-01-10

When you query this table, SQLite will return some or all of these columns depending on what you ask for.

Basic SQL syntax

SQL statements are made of keywords and names. Keywords like SELECT and FROM tell SQLite what action to perform. Names like employees tell it which table to use.

SQL keywords are not case-sensitive, but it is common to write them in uppercase because they are easier to spot. Statements are often ended with a semicolon: 

SELECT * FROM employees;

What does SELECT do?

The SELECT command reads data from a table. It does not change the table; it only displays results.

  • SELECT says you want to retrieve data
  • * means “all columns”
  • FROM employees says the data should come from the employees table

If you only want a few columns, you can list them by name: 

SELECT name, department FROM employees;

This returns only the name and department columns from every row in the employees table.

The order of the columns in your SELECT list does not have to match the order of the table. You can request columns in whatever order you want, such as SELECT department, name FROM employees;, and the results will follow that order.

Exercise

You have just seen that the employees table contains columns like id, name, department, salary, and hired_date. To display every column for every employee, use: 

SELECT * FROM employees;

Now type that query yourself and run it. You should see all rows and all columns from the table.
