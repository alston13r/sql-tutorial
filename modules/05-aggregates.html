Aggregate Functions

All exercises in this lesson have been completed.

So far, your queries return individual rows. Aggregate functions summarize many rows into a single value — for example, how many employees there are, or what the average salary is.

Recap: rows vs summaries

A normal SELECT lists rows from a table. An aggregate query returns one result row that describes the whole set, such as a count or an average.

The practice database has 14 employees. Frank Lee has a NULL salary, which matters for some aggregates — COUNT(*) still counts his row, but COUNT(salary) does not.

COUNT()

COUNT() tells you how many rows match. COUNT(*) counts every row; COUNT(column) counts only rows where that column is not NULL.

To count all employees: 

SELECT COUNT(*) FROM employees;

In this database, COUNT(*) returns 14 and COUNT(salary) returns 13 because Frank Lee's salary is unknown.

SUM()

SUM() adds up values in a numeric column. Rows where the column is NULL are skipped.

To find total payroll for all employees: 

SELECT SUM(salary) FROM employees;

AVG()

AVG() calculates the average of a numeric column, again ignoring NULL values.

To find the average salary company-wide: 

SELECT AVG(salary) FROM employees;

MIN() and MAX()

MIN() and MAX() return the smallest and largest values in a column. They work on numbers, dates, and text.

To find the lowest and highest salary: 

SELECT MIN(salary), MAX(salary) FROM employees;

To get averages or totals per department, you will use GROUP BY in a later module. For now, aggregates run over the whole table (or whatever rows your WHERE clause selects).
