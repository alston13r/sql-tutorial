Recap: filtering rows

Exercises in this module return employee name values while filtering on name, department, or first-name ranges. The practice database has 14 employees across several departments.

LIKE and Wildcards

LIKE matches text patterns. % stands for any number of characters (including none); _ stands for exactly one character.

In SQLite, LIKE is case-insensitive for ASCII letters: '%lee%', '%Lee%', and '%LEE%' all match Frank Lee. That is different from =, which is case-sensitive — name = 'frank lee' would not match Frank Lee.

To find names containing Lee: 

SELECT name FROM employees WHERE name LIKE '%Lee%';

Result: Frank Lee

IN

IN checks whether a value appears in a list. It is often clearer than writing several OR conditions.

To find employees in a few departments: 

SELECT name FROM employees
WHERE department IN ('Engineering', 'Finance', 'Legal');

BETWEEN

BETWEEN selects values within a range, inclusive of both endpoints. It works on text as well as numbers — strings are compared in alphabetical order.

To find names from Alice Chen through David Kim: 

SELECT name FROM employees
WHERE name BETWEEN 'Alice Chen' AND 'David Kim';

Comparison uses the full name string alphabetically, and both boundary names are included in the result.
