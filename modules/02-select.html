SELECT Basics

All exercises in this lesson have been completed.

You already know how to read data with SELECT. This module adds three common patterns for shaping query results: removing duplicates, renaming columns, and limiting how many rows you get back.

Recap: reading data

A basic query selects columns from a table: 

SELECT name, department FROM employees;

This returns every row from employees, showing only the columns you listed.

SELECT DISTINCT

Sometimes a column contains duplicate values. In the employees table, multiple people work in the same department. If you run: 

SELECT department FROM employees;

you will see Engineering and Sales more than once.

To list each department only once, add DISTINCT: 

SELECT DISTINCT department FROM employees;

The result contains four unique departments: Engineering, Marketing, Sales, and HR.

Aliases

An alias gives a column (or table) a temporary name in the query result. This is useful when you want clearer or shorter column headings.

Use the AS keyword to rename a column: 

SELECT name AS employee_name, department AS dept
FROM employees;

The output columns will be labeled employee_name and dept instead of name and department. The underlying table is not changed.

LIMIT

The LIMIT clause restricts how many rows a query returns. In SQLite, you use LIMIT where some other databases use TOP.

To return only the first three rows: 

SELECT * FROM employees LIMIT 3;

This gives you three rows from the result set. Sorting with ORDER BY comes in a later module; for now, think of LIMIT as capping the number of rows you see.
