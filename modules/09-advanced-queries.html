Advanced Queries

All exercises in this lesson have been completed.

This module introduces three tools for more expressive SQL: UNION for combining result sets, subqueries with EXISTS for checking related rows, and CASE for conditional output.

UNION and UNION ALL

UNION stacks the results of two compatible queries into one result set and removes duplicate rows. UNION ALL keeps duplicates. Both sides must return the same number of columns in the same order.

To list employees in Engineering or Finance: 

SELECT name FROM employees WHERE department = 'Engineering'
UNION
SELECT name FROM employees WHERE department = 'Finance';

UNION ALL would keep duplicate rows if the same result appeared in both queries.

Subqueries and EXISTS

A subquery is a query inside another query. EXISTS checks whether that inner query returns at least one row. When only existence matters, SELECT 1 is conventional.

To list departments that have at least one employee: 

SELECT d.name
FROM departments d
WHERE EXISTS (
  SELECT 1
  FROM employees e
  WHERE e.department_id = d.id
);

The inner query runs for each department row. If it finds at least one matching employee, EXISTS is true.

CASE

CASE adds conditional logic inside a query. SQL checks each WHEN branch from top to bottom and returns the first match.

To label salaries by band: 

SELECT name,
       CASE
         WHEN salary >= 90000 THEN 'High'
         WHEN salary >= 70000 THEN 'Medium'
         ELSE 'Low'
       END AS salary_band
FROM employees;

The order matters: once a WHEN branch matches, later branches are skipped.

Summary

These advanced query tools solve different kinds of problems:

  • UNION combines compatible result sets into one output table.
  • EXISTS keeps or removes rows based on whether a related subquery returns anything.
  • CASE builds conditional labels or values inside a query result.

A good rule of thumb: use UNION when you need to stack results, EXISTS when you need to ask “does a related row exist?”, and CASE when you need SQL to label rows differently based on conditions.
